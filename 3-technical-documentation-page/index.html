What is Static Analysis

It is tooling that scrutinizes code without running it, as opposed to "dynamic analysis" which tests the result of running code - like testing (unit tests or large end-to-end tests).

What is Formatting

A formatter is a tool that

  1. reads your source code
  2. ignores your formatting
  3. rewrites it based on config rules

Ex. Prettier

What is a Linter

A linter is a tool that runs a set of checks on your source code

  • Each check is a rule
  • Each rule may report on code it doesn't like
  • Each complaint may contain an optional autofix

ESLint is a tool built into most text editors for identifying and reporting on patterns found in ECMAScript/JavaScript code.

  • Every single rule of ESLint is a plugin
  • More rules can be added at runtime
  • The functionality of ESLint can be extended further by using community plugins, configs and parsers
Running ESLint

To install ESLint in a Node.js app, enter in your terminal: 


				npm install eslint --save-dev
				npm init @eslint/config # eslint config wizard

				npx eslint . # lint everything, in this directory

Note: npm init @eslint/config assumes you have a package.json file already. If you donâ€™t, make sure to run npm init or yarn init beforehand.

Configuring ESLint

ESLint rules can be configured in .eslintrc.js. 


				// .eslintrc.js
				module.exports = {
					// what eslint team thinks most codebases probably want
					"extends": [
						"eslint:recommended",
						// config for React
						"plugin:react/recommended",
						"plugin:react-hooks/recommended"
					],
					"parserOptions": {
						"ecmaFeatures": {
							"jsx": true
						}
					},
					"plugins": ["react", "react-hooks"],
					"settings": {
						"react": {
							"version": "detect"
						}
					}
					
				}

Bonus-er Round

lint:fix will fix whatever stuff has an autofix available

--max-warnings 0 silences warnings from eslint, which is good to train us to not become complacent about eslint messages. 


				// package.json
				{
					"scripts": {
						"lint": "eslint . --report-unused-disable-directives",
						"lint:fix": "npm run lint -- --fix" 
					}
				}
